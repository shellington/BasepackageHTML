Website Image Optimisation using GTmetrix

Written by Admin on 10 October 2014
Tags: Development Category: Web Development
Comments: 0 comments Comment

Images can cause a vast array of website page speed issues, as they take up the majority of a website’s bandwidth. But this doesn’t need to be an issue, thanks to some handy free tools which can analyse page load speed and performance.

Here at Ph.Creative, we make use of lots of tools to enable us to take a closer look at how quickly a web page loads, identify why this is happening and take action to improve page load speed, one of which is our personal favourite, GTmetrix.

GTmetrix is a free tool that grades your website’s performance. It can analyse roughly 30 different actions to take in order to improve website performance and load time. It uses Google Page Speed and Yahoo! YSlow to grade your site's performance and is great for pointing out recommendations for improving and optimising images on your website.

Optimise images

The best bit about GTmetrix is the ‘optimise images’ section, where it lists all images on the page you’re testing that can be compressed.

Not only does it tell you how much your image files size can be reduced by, it also gives you an optimised version of the image for you to save and replace the original. How easy is that?

Specify image dimensions

It’s important to always ‘specify image dimensions’ in your markup within your <img> tags.Getmix3This is because, when the browser renders a page, it needs to be able to flow around replaceable elements such as images. It can begin to render a page even before images are downloaded, provided that it knows the dimensions to wrap non-replaceable elements around.

If no dimensions are specified in the containing document, or if the dimensions specified don't match those of the actual images, the browser will require a reflow and repaint once the images are downloaded. This slows the page and potentially leads to elements moving around during download.

Serve scaled images

When displaying images on your website it’s important to serve the right size images. Sounds simple, but that’s not always the case when building mobile-first, responsive websites. There’s no supported solution for responsive images yet (though the proposed HTML5 Picture element could be a solution in the future).
